How it works

Up and running in 60 seconds

1

Install globally

npm install -g @memcode/cli
2

Initialize your repo

cd my-project && memory init --hooks

Creates .memory/memory.db and installs git hooks automatically.

3

Work normally

Commit code as usual. Every commit silently saves a checkpoint with branch, git SHA, and context summary.

4

Auto-inject into your AI

memory copilot setup

Writes context to .github/copilot-instructions.md and CLAUDE.md. Every new chat in VS Code Copilot or Claude Code starts with your full project memory — automatically refreshed after every checkpoint.

5
Pro

Sync across machines

memory sync auth && memory sync push

Pull on any other machine with memory sync pull.

FAQ

Common questions

Is my code or conversation content uploaded to the cloud?

No. MemCode only stores structured metadata: checkpoint summaries, decision records, and task titles. No raw code, no conversation transcripts. And even that metadata is encrypted client-side before leaving your machine.

What Node.js version do I need?

Node.js ≥ 22.15.0. MemCode uses Node's built-in SQLite module (node:sqlite), which was stabilised in 22.15. No native compilation required.

Can I use MemCode without cloud sync?

Absolutely. The free tier is fully featured — unlimited local memory, git hooks, recall, and context packs. Cloud sync is purely additive.

How does the encryption work?

Your encryption key is derived from your passphrase + workspace ID using SHA-256, entirely on your machine. We store AES-256-GCM blobs and have no mathematical way to decrypt them without your passphrase.

Can I export my data?

Yes. Your primary data lives in .memory/memory.db — a standard SQLite file you own completely. You can inspect, export, or migrate it any time.

What happens if I cancel Pro?

Your local memory is unaffected — it never depended on the cloud. Cloud blobs are retained for 30 days after cancellation so you can re-activate or export.
